ARINC 429
Embedded Systems · Avionics · Real-World Debugging

A real-world deep dive into ARINC 429 from implementation perspective — not theory. Covers word structure, decoding, BNR vs BCD pitfalls, and actual avionics debugging lessons.

// blog content

ARINC 429 is a unidirectional avionics data bus used across aircraft systems.

Every transmission is a 32-bit word containing:

  • Label (8 bits — bit reversed)
  • SDI (2 bits)
  • Data (19 bits)
  • SSM (status/sign)
  • Parity

The first major mistake engineers make is label decoding.

Labels are transmitted LSB-first, meaning they must be bit-reversed before comparison.

Second major trap: BNR vs BCD.

  • BNR → binary fixed-point (scaled)
  • BCD → decimal digits packed in 4-bit groups

If you decode BCD as BNR, you’ll get values that look correct — but are wrong.

SSM (Sign/Status Matrix) is critical:

  • Normal Operation
  • Failure Warning
  • No Computed Data

If SSM ≠ valid → ignore the data.

In real avionics systems, ignoring SSM leads to dangerous UI bugs where stale or invalid values are shown.

Production Lessons:

  • Always reverse label bits
  • Always validate parity
  • Always check SSM before using data
  • Always apply correct resolution (LSB weight)

ARINC 429 is old, but once understood — it's extremely deterministic and reliable.
